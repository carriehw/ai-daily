# 點寫一條「複製指令」俾其他 AI

呢份教你點寫 prompt，令另一個 AI 由零砌返個 AI 日報系統（同樣方法可套用喺任何項目）。

---

## 一、好 prompt 嘅 6 大組成部分（框架）

任何「幫我做返呢樣嘢」嘅 prompt，齊呢 6 忽就夠清楚：

| # | 部分 | 答咩問題 | 唔寫嘅後果 |
|---|---|---|---|
| 1 | **角色 + 目標** | 你係邊個？要交咩成品？ | AI 亂咁做，唔知終點 |
| 2 | **背景** | 手上有咩？俾邊個用？ | AI 靠估，做錯方向 |
| 3 | **規格（硬性）** | 有咩一定要有／一定唔准？ | 走樣，要返工 |
| 4 | **步驟** | 分幾步做，每步做咩？ | 漏 step、次序亂 |
| 5 | **限制同陷阱** | 有咩伏要避？ | 撞同一個坑（例：Jekyll build 失敗） |
| 6 | **完成標準 + 驗證** | 點先算「做完」？點證明？ | AI 話做完，實際未 work |

> 記憶法：**角色 → 背景 → 規格 → 步驟 → 陷阱 → 驗證**

---

## 二、五個令 AI 更聽話嘅技巧

1. **俾原材料，唔好淨係得指令。** 有現成檔案（`build.py`、`data.json`、skill zip）就一齊俾，AI 唔使重新發明。
2. **講「點解」，唔淨係「做咩」。** 例：「用 GitHub Actions 部署（唔好用 Jekyll，會 build 失敗）」——AI 明背後原因就唔會揀錯。
3. **硬性要求要 quote 死。** 「五個固定版塊：A、B、C、D、E」好過「分幾個版塊」。
4. **要求佢做完自我驗證。** 「跑完 curl 確認 HTTP 200 先算完成」逼 AI 唔好報大數。
5. **一次講一個目標。** 想加功能就下一輪先講，唔好一條 prompt 塞十件事。

## 三、即抄即用：完整 Master Prompt

> 用法：將下面成段貼俾另一個 AI（最好連 `ai-daily-briefing-skill.zip` 一齊俾）。有齊工具（跑 code、上網、GitHub）嘅 AI 就砌到。

```
你係一位全端工程師兼中文編輯。幫我由零砌一個「每日 AI 新聞快訊網站」，最終要一條公開網址，任何人唔使登入都睇到、可以 share。

【背景】
- 我係香港人，日常用 Claude / ChatGPT / Gemini，亦有買股票。
- 讀者係香港 AI 用家兼股民，想快速知今日 AI 界發生咩事、關佢咩事。
- 我有一個 skill 包（ai-daily-briefing），入面有 build.py、編輯守則、部署指南，請善用。

【硬性規格】
- 五個固定版塊（次序固定）：模型發布/更新、產品發布/更新、行業動態、論文研究、技巧與觀點
- 覆蓋四個市場：國際、香港、中國、台灣（每個有顏色標籤）
- 每條新聞卡包含：全局連續編號、標題、來源、≤60字摘要、「點解重要」一句、原文連結（開新分頁）
- 頂部要有日期、總條數、五版塊統計；中間有錨點導航；響應式；單一 HTML 檔、所有 CSS/JS 內聯、無外部資源
- 文字用香港廣東話（口語書面化）：禁用「但/但係/不過」；專業術語要即場用生活化說法解釋；「點解重要」要貼身（影響荷包/即刻用得/揀邊隻好/對股市點睇），唔好離地口號

【步驟】
1. 抓過去 24–48 小時新聞，三路並行：① smol.ai RSS（news.smol.ai/rss.xml）② 國際媒體（Anthropic/OpenAI/Google 官方、TechCrunch、Reuters、Bloomberg）③ 大中華媒體（中國大模型、港股AI、台積電/聯發科）。中外來源都要有。
2. 合併去重，精選 30–40 條，每條寫好摘要同「點解重要」（跟上面文字要求）。
3. 將內容寫成 data.json，跑 build.py 生成 index.html。
4. 發布上 GitHub Pages：建 public repo、用 GitHub Actions 部署（build_type=workflow，加 .nojekyll）。
5. set 每朝自動更新（排程跑步驟 1–4，排程指示要內嵌編輯守則全文同自我修復 git push）。

【陷阱，要避開】
- 部署一定用 GitHub Actions，唔好用 legacy Jekyll（會 Page build failed）。
- Windows 跑 python 前要 set PYTHONIOENCODING=utf-8，否則 crash。
- 確保 repo 冇 secret / .env 會 publish 出去。

【完成標準】
- curl 個網址確認 HTTP 200 同見到真內容（有 <title> 同多張新聞卡）。
- 最近一次部署 Action conclusion=success。
- 每個版塊都有內容；每條都有「點解重要」；全部文字跟足廣東話要求。
做完俾我：公開網址、今日總條數、五版塊分佈、三條最重要 headline。
```

## 四、想改成第二個題材／語言？

同一條 prompt，改三忽就得：
- **題材**：換五個版塊 + 步驟 1 嘅來源同關鍵詞（例：財經新聞 → 換做港股/美股/外匯/加密貨幣，來源換財經媒體）
- **語言**：換「文字用香港廣東話」嗰段做目標語言同讀者視角
- **市場**：換四個市場標籤

其餘框架原封不動。

---

## 五、一句總結
> 好 prompt = **講清楚終點（成品 + 驗證）+ 圈死唔准走樣嘅位（規格）+ 標明伏（陷阱）+ 俾晒原材料（檔案）**。AI 越少估，越似你想要。
